# Aman Team Workflows

## standup
When someone says "standup" or at the scheduled standup time:
1. Initiate — greet the team and start the round. "Good morning team! Standup time."
2. Collect — ask each team member (from memory) three questions:
   - What did you accomplish yesterday?
   - What are you working on today?
   - Any blockers?
3. Track — log each person's update to memory.
4. Summarize — after everyone responds, post a structured summary:
   - Each member's yesterday/today in one line
   - Blockers highlighted with owner
   - Action items from blockers
5. Follow-up — if someone doesn't respond within 30 minutes, skip them (async-friendly).
6. Save — store standup data and any new blockers to memory.

## decision
When the team makes a decision or someone says "let's decide" or "we decided":
1. Capture — what was decided? Ask for clarity if ambiguous.
2. Context — who was present? What were the alternatives considered?
3. Rationale — why this choice? What factors drove it?
4. Log — save as decision memory with: decision, date, participants, alternatives, rationale.
5. Confirm — post the formatted decision back to the group for verification.
6. Link — connect to related project or previous decisions in memory if relevant.

## project
When someone mentions a new project or says "new project" / "project update":
1. If new: gather name, owner, team members, deadline, scope, status (not started/in progress).
2. If update: recall project from memory, update status, note what changed.
3. Store — save project details to memory.
4. Summary — post current project status back to the group.
5. Alert — if deadline is within 7 days and status isn't "in review" or "done", flag it.

## blocker
When someone reports a blocker or says "blocked" / "stuck":
1. Capture — what's the blocker? Which project? Who's affected?
2. Identify — who can help unblock? (Check team member skills in memory)
3. Log — save blocker to memory with date raised.
4. Notify — "Looks like [Name] might be able to help with this. [Name], can you take a look?"
5. Track — if unresolved after 3 days, remind the team.
6. Resolve — when resolved, update memory and note the solution.

## onboarding
When a new person joins the group or someone says "onboard [name]":
1. Welcome — greet the new member warmly.
2. Team — list all team members with their roles and what they work on.
3. Projects — summarize active projects: name, status, owner, deadline.
4. Decisions — list the 10 most recent key decisions with brief context.
5. Stack/Tools — what technologies, tools, and processes does the team use?
6. Norms — any team preferences (standup time, communication style, working hours)?
7. Save — store new team member profile to memory.

## meeting
When someone says "meeting notes" or "let's capture this":
1. Topic — what was discussed?
2. Attendees — who was present?
3. Summary — key points discussed (bullet points).
4. Decisions — any decisions made? (Route to decision workflow)
5. Actions — action items with owner and deadline.
6. Save — store meeting notes to memory.
7. Post — share formatted notes in the group.

## report
When someone asks "weekly report" or "team status" or "how are we doing":
1. Period — which period? Default to current week.
2. Projects — status of all active projects with progress.
3. Decisions — decisions made this period.
4. Blockers — current blockers and their age.
5. Standups — participation rate (who showed up consistently).
6. Completed — what was shipped/finished this period.
7. Next — what's the focus for next period?
