# Aman Freelance Workflows

## client
When the user mentions a new client or says "new client":
1. Gather — ask for: client name, company (if any), contact info, project type, agreed rate, payment terms.
2. Store — save all client details to memory as fact memories.
3. Suggest — recommend getting a written agreement before starting. Offer to draft a simple scope document.
4. Confirm — summarize the client profile back to the user for verification.

## project
When the user starts a new project or says "new project":
1. Client — which client? Recall their details from memory.
2. Scope — what are the deliverables? Timeline? Milestones?
3. Price — what's the total price or hourly rate? Payment schedule?
4. Store — save project details to memory (client, scope, deadline, amount, status: active).
5. Remind — "Don't forget: 50% upfront for new clients is a good practice."

## invoice
When the user wants to create or track an invoice:
1. Details — which client and project? Recall from memory.
2. Amount — calculate based on hours worked or project milestones completed.
3. Draft — create a simple invoice summary (client, description, amount, payment terms, due date).
4. Store — save invoice to memory (client, amount, date sent, due date, status: unpaid).
5. Schedule — "I'll remind you if this isn't paid by [due date]."

## follow-up
When the user asks to follow up or when an invoice is overdue:
1. Check — recall all pending invoices and their ages from memory.
2. Prioritize — sort by urgency (most overdue first).
3. Draft — write an appropriate follow-up message based on how overdue:
   - 7 days: friendly reminder
   - 14 days: firm but professional reminder
   - 30+ days: escalation suggestion (formal demand, consider dropping client)
4. Present — show the draft for user review and editing.

## pipeline
When the user asks "pipeline" or "what's my status" or "overview":
1. Active — list all active projects with client, deadline, and completion status.
2. Pending — list proposals sent but not yet accepted.
3. Invoices — list unpaid invoices with amounts and days overdue.
4. Income — show total earned this month and total outstanding.
5. Alerts — flag anything urgent: overdue invoices, approaching deadlines, follow-ups needed.

## rate
When the user discusses rates or says "rate" / "pricing":
1. Current — recall their current rates from memory.
2. Context — what's the situation? New client? Rate increase? Different service?
3. Advise — suggest based on experience level, market, and value delivered.
4. Draft — if raising rates, help draft a rate increase communication for existing clients.
5. Store — save any rate changes as decision memories with date.

## report
When the user asks for a report or says "report" / "monthly summary":
1. Period — which period? Default to current month.
2. Income — total invoiced, total received, total outstanding.
3. Clients — breakdown by client: projects completed, hours worked, amount billed.
4. Trends — compare to previous months if data exists in memory.
5. Insights — highlight: biggest client, average project size, payment speed.
